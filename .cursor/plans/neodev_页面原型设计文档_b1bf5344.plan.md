## 三、文档建议结构

### 1. 设计规范（Design Tokens）

- **色彩**：背景 `#0D0D12`、主交互 `#00F0FF`、AI 洞察 `#B026FF`、告警 `#FF0055`；并补充中性灰、边框、禁用态、悬浮态等衍生色。
- **字体**：标题/正文字号与字重，可选字体族（如科技感无衬线）。
- **圆角与阴影**：卡片、输入框、按钮的圆角与发光/阴影规则。
- **间距系统**：4/8/16/24/32 等基准，用于栅格与组件内边距。

### 2. 全局布局与壳（Shell）

- **三栏比例**：左侧 15%、中部 60%、右侧 25%；注明是否支持折叠（如 Agent 可收起到图标栏）。
- **左侧系统导航**：导航项与 V1.0 拓扑对应（入口首页、资产接入、图谱重构、版本驾驶舱及其子入口）；图标 + 文案；高亮与展开规则。
- **右侧 Agent 视窗**：常驻、最小高度、滚动；上下文同步原则（随路由/选中需求/选中提交变化）；典型状态：迎新、解析播报、智能建议、根因告警。
- **路由与导航映射表**：V1.0 页面 → 建议 path（与现有 `/projects`、`/impact`、`/repos` 等的对应或新增 path）。

### 3. 页面拓扑图（可选 Mermaid）

- 用流程图复现你提供的“从入口到三层”的拓扑，并标注每个节点对应文档内的小节号，便于跳读。

### 4. 逐页详细设计（核心）

对以下四页，每页按**统一模板**展开：

- **页面目标与用户故事**（1～2 句）。
- **区域划分**：中栏（及必要时中栏内左右分栏）与右侧 Agent 的职责。
- **组件树/清单**：从区域到具体控件（如“发光输入框”“流光按钮”“树状列表”“详情抽屉”“时间轴”“拓扑图”等），并标注是否复用。
- **状态与反馈**：加载态、空态、错误态、成功态（如“校验并接入资产”成功后的流光反馈）。
- **交互说明**：点击、拖拽、跳转（含 Tab A ↔ Tab B 双向跳转）、与 Agent 的联动。
- **文案与示例**：关键按钮、提示、Agent 话术（直接引用你提供的示例或扩展 1～2 条）。
- **数据/API 触点**：本页涉及的实体与操作（如 projects、repos 校验、versions、parse 触发、requirements、commits、impact_analyses、Neo4j 查询）；不写具体 API 路径，只写“对接点”与引用数据结构文档。

**四页清单**：

| 序号 | 页面名称 | 对应 V1.0 描述 |
|------|----------|----------------|
| 1 | 项目初始化与资产接入 | 填写项目元数据、绑定 Git、校验接入；Agent 迎新 |
| 2 | 版本基线与图谱构建 | 选择分支/版本、启动图谱重构、进度雷达；Agent 解析播报 |
| 3 | 版本驾驶舱 - Tab A 需求流转矩阵 | 需求树、详情抽屉、代码追溯区、关联提交；Agent 智能建议 |
| 4 | 版本驾驶舱 - Tab B 提交与影响面分析 | Commit 时间轴、动态影响面拓扑图、业务归属标签；Agent 根因与风险 |

### 5. 组件复用与优先级

- 列出跨页复用组件（如发光输入框、流光按钮、抽屉、时间轴、拓扑图容器），并标注实现优先级（P0 与现有流程强相关 / P1 增强体验）。

### 6. 与 PRD/数据模型映射（简表）

- **路由**：原型页面 → 现有或建议 path。
- **实体**：页面 ↔ projects、versions、requirements、commits、impact_analyses、Neo4j 图。
- **流程**：与 PRD 2.1 主流程（选择项目→版本→需求→提交→影响面）的对应关系。

### 7. 后续技术设计可选延伸

- 在文档末尾用一小节说明：可根据需要拆出三份技术详细设计——**A. 核心图模型**（Neo4j 节点与关系定义）、**B. 自动绑定算法流**（Commit↔需求 的 RAG/规则推断）、**C. 前端图组件与 Agent 通信**（G6/ECharts 选型、Context 同步的 JSON 结构）。本原型文档不展开 A/B/C 的实现细节，仅引用本节。
